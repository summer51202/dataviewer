## 建議閱讀順序

1. 先看 [UI Mockup](./ui-mockup.md)
2. 再看 [功能 Workflow](./functional-workflow.md)
3. 再看 [模組架構圖](./module-architecture.md)
4. 接著確認 [Software Stack](./software-stack.md)
5. 最後依照 [實作計畫](./implementation-plan.md) 開始拆任務

## 文件定位

- `docs/user-guide-zh-TW.md`：給使用者看的產品目的、主要頁面與使用流程導覽
- `spec.md`：產品規格與 MVP 定義
- `docs/ui-mockup.md`：畫面結構與互動草圖
- `docs/module-architecture.md`：系統分層、模組責任與資料流
- `docs/functional-workflow.md`：使用者操作與系統處理流程
- `docs/software-stack.md`：建議技術選型與採用理由
- `docs/implementation-plan.md`：分階段實作順序與交付目標
- `docs/release-notes-v*.md`：各版本功能更新與修正紀錄
